\doxysubsection{Detailed Description}
Na classe \char`\"{}\+Cliente\char`\"{}, a qual possui como classe amiga \char`\"{}\+Locadora\char`\"{}, existem como atributos protegidos o nome do cliente (\char`\"{}string nome\char`\"{}) e seu cpf (\char`\"{}long long cpf\char`\"{}). A classe também contém como atributos públicos um construtor que recebe o cpf e o nome do cliente (\char`\"{}\+Cliente(long long cpf, string nome)\char`\"{}) um destrutor (\char`\"{}$\sim$\+Cliente()\char`\"{}) e os pontos dos clientes com base nos filmes alugados (\char`\"{}int pontos\char`\"{}). Existe também um método para identificação de clientes (\char`\"{}void ler\+Cliente()\char`\"{}), um método para controlar a quantidade e quais filmes foram alugados (\char`\"{}void alugar (\+Filme$\ast$ filme)\char`\"{}), um método para controlar a quantidade e quais filmes foram devolvidos (\char`\"{}void devolver ()\char`\"{}), um método para recomendar filmes aos clientes (\char`\"{}void recomendar(vector $<$\+Cliente$\ast$$>$ clientes)\char`\"{}), um método para calcular a quantidade de filmes similares alugados pelos clientes (\char`\"{}int calcular\+Similaridade (\+Cliente$\ast$ cliente)\char`\"{}), um método para definir quais clientes possuem gostos similares (\char`\"{}void definir\+Similares (vector $<$\+Cliente$\ast$$>$ clientes)\char`\"{}), um vetor de ponteiros para controlar quais filmes foram alugados (\char`\"{}vector$<$\+Filme$\ast$$>$ filmes\+Alugados\char`\"{}) um vetor de ponteiros para adicionar filmes ao histórico dos clientes (\char`\"{}vector$<$\+Filme$\ast$$>$ historico\char`\"{}), um vetor de ponteiros de filmes recomendados (\char`\"{}vector$<$\+Filme$\ast$$>$ recomendados\char`\"{}), um vetor de ponteiros de filmes similares (\char`\"{}vector$<$\+Cliente$\ast$$>$ similares\char`\"{}) e, por fim, um vetor de ponteiros para recomendar os filmes aos clientes com base na similaridade entre os gostos dos mesmos (\char`\"{}vector $<$\+Filme$\ast$$>$ recomendar\+Por\+Similar(\+Cliente$\ast$ cliente)\char`\"{}).

{\footnotesize\ttfamily void Cliente\+::definir\+Similares (\begin{DoxyParamCaption}\item[{vector$<$ \mbox{\hyperlink{classCliente}{Cliente}} $\ast$ $>$}]{clientes }\end{DoxyParamCaption})}



Método \char`\"{}definir\+Similares\char`\"{} da classe \char`\"{}\+Cliente\char`\"{}, utilizado para fins de recomendação. 

Esse método recebe um vetor de clientes e, a partir daí, realiza uma filtragem, em busca de selecionar os 3 usuários com maior similaridade com o cliente atual, e colocá-\/los no seu atributo \char`\"{}similares\char`\"{}. Caso esse vetor parâmetro possua menos que três clientes, essa comparação não precisa ser realizada, e os clientes já são alocados nos similares imediatamente. Caso haja mais de 3, a seleção prossegue. Para cada cliente do parâmetro, é calculado o grau de similaridade, e comparado com os 3 clientes mais similares. Se o grau de similaridade for maior que algum, esse ranking é reformulado para encaixar o novo cliente similar. O cliente com maior similaridade fica em primeiro lugar e assim por diante. Depois de percorrer todo o vetor, os 3 clientes são adicionados ao vetor \char`\"{}similares\char`\"{} de acordo com o ranking previamente feito.

Método \char`\"{}recomendar\char`\"{} da classe \char`\"{}\+Cliente\char`\"{}, utilizado para fins de recomendação. 

Esse método recebe como parâmetro um vetor de clientes e retorna, a partir deles, um vetor de filmes recomendados. Para isso, ele começa definindo os clientes de gosto mais similar ao cliente atual, pelo método \char`\"{}definir\+Similares\char`\"{}. Se não, é utilizado o método \char`\"{}recomendar\+Por\+Similar\char`\"{} para que cada um dos clientes no vetor \char`\"{}similares\char`\"{} forneça um vetor de filmes, que se juntarão em um só vetor de filmes recomendados, que fica no atributo \char`\"{}recomendados\char`\"{}.
